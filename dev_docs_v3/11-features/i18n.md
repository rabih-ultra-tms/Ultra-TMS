# Internationalization (i18n)

> Source: `dev_docs/10-features/78-i18n-standards.md`
> Last updated: 2026-03-07

---

## Overview

Ultra TMS supports multi-language interfaces with **Spanish as priority #1** (significant Hispanic driver population in US trucking). Uses `next-intl` on the frontend and `nestjs-i18n` on the backend.

---

## Supported Locales

| Locale | Language | Priority | Notes |
|--------|----------|----------|-------|
| `en-US` | English (US) | Default | All portals |
| `es-MX` | Spanish (Mexico) | High | Driver portal, Carrier portal |
| `es-US` | Spanish (US) | High | Fallback for es-MX |

### Future Phases

- `fr-CA` — French Canadian (Phase D: Cross-border)
- `pt-BR` — Portuguese Brazil (Phase E: Expansion)

---

## Core Rules

1. **NEVER hardcode user-facing strings** — Always use translation keys
2. **Spanish is priority #1** — Required for Hispanic driver population
3. **All dates, times, currencies, numbers must be locale-aware**
4. **Driver mobile app MUST support Spanish from day one**

## Frontend: next-intl Setup

```typescript
// lib/i18n.ts
import { getRequestConfig } from 'next-intl/server';

export default getRequestConfig(async ({ locale }) => ({
  messages: (await import(`../messages/${locale}/common.json`)).default,
}));
```

### Usage in Components

```tsx
import { useTranslations } from 'next-intl';

export function CarrierList() {
  const t = useTranslations('carriers');

  return (
    <div>
      <h1>{t('title')}</h1>
      <p>{t('empty', { count: 0 })}</p>
      <Button>{t('actions.create')}</Button>
    </div>
  );
}
```

## Locale-Aware Formatting

```typescript
// Dates
const formatted = new Intl.DateTimeFormat(locale, {
  year: 'numeric', month: 'short', day: 'numeric',
}).format(date);

// Currency
const amount = new Intl.NumberFormat(locale, {
  style: 'currency', currency: 'USD',
}).format(value);

// Numbers
const weight = new Intl.NumberFormat(locale).format(42000);
// en-US: "42,000"  |  es-MX: "42,000"

// Relative time
const rtf = new Intl.RelativeTimeFormat(locale, { numeric: 'auto' });
rtf.format(-1, 'day'); // "yesterday" / "ayer"
```

## Backend: nestjs-i18n

```typescript
// Error messages
throw new BadRequestException(
  this.i18n.translate('errors.carrier.notFound', { lang: user.locale })
);

// Validation messages
@IsNotEmpty({ message: i18nValidationMessage('validation.required') })
name: string;
```

---

## Translation Key Naming Convention

```
{module}.{section}.{key}

carriers.title          → "Carriers"
carriers.fields.name    → "Carrier Name"
carriers.actions.create → "Add Carrier"
common.save             → "Save"
common.cancel           → "Cancel"
errors.notFound         → "Resource not found"
```
