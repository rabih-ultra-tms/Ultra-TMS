üîê Security Consideration

Permission-Aware Search Results

Search service must filter results based on the user's access to underlying entities:

  • Example: If user can't access carrier banking info, search shouldn't return those records
  • Solution: Post-filter search results by checking entity-level permissions
  • Alternative: Pre-index with permission tags, filter at query time
// Recommended approach
const results = await searchService.search(query);
const filteredResults = await permissionService.filterByAccess(
  results,
  user.roles,
  user.tenantId
);

üí° Recommendations

  1. P1 Add permission-aware result filtering
    • Check user's access to each entity type in results
    • Hide sensitive fields from search results
  2. P1 Add entity-type RBAC - Users should only search entity types they can access.
  3. P2 Add Elasticsearch integration - Current DB-based search won't scale.
  4. P2 Add fuzzy matching - Handle typos.
  5. P3 Add search analytics - Track common searches for optimization.
