üîê Security Architecture

Configuration Security Model

Config service implements a layered security approach:

  • Secret values: Marked with isSecret: true, never returned in API responses
  • System configs: SUPER_ADMIN only modification
  • Tenant configs: Tenant ADMIN can modify their own settings
  • Public configs: Non-sensitive settings available to authenticated users
  • History tracking: All changes logged with old/new values and user ID
